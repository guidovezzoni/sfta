# 1.2 Dashboard UI

## User Story

**As a** rider glancing at the motorcycle dashboard,
**I want** a dark, high-contrast landscape display showing speed, battery, power, ride settings, and warnings,
**So that** I can read my bike's telemetry at a glance while riding.

## Scope

### Phase 4: Theme

- **`ui/theme/Color.kt`**: replace with dashboard palette -- `DashboardBackground` (#0D0D0D), `DashboardSurface` (#1A1A1A), `StarkGold` (#C8A84E), semantic colours: `SpeedWhite`, `BatteryGreen`/`Amber`/`Red`, `WarningAmber`/`Red`, `PowerBlue`
- **`ui/theme/Theme.kt`**: dark-only `darkColorScheme`, no dynamic colours, remove `darkTheme`/`dynamicColor` params
- **`ui/theme/Type.kt`**: add `displayLarge` (96sp bold for speed), `titleLarge` (20sp for panel headers), `labelMedium` (12sp for labels)

### Phase 5: MVI Contract + ViewModel

- **`ui/state/DashboardUiState.kt`**: flat data class with all fields + defaults -- `isLoading`, `errorMessage`, `currentSpeedKmh`, battery fields, motor fields, ride settings fields, `warnings`, `bikeName`
- **`ui/intent/DashboardUiIntent.kt`**: `LoadDashboard`, `RetryLoad`
- **`ui/effect/DashboardUiEffect.kt`**: empty sealed class (structural placeholder)
- **`ui/viewmodel/DashboardViewModel.kt`**: processes intents, calls use case, flattens domain model to UiState
- **`ui/viewmodel/DashboardViewModelFactory.kt`**: `ViewModelProvider.Factory` for manual DI

### Phase 6: Composables

- **`ui/screens/DashboardScreen.kt`**: main screen composable -- `when` block: `isLoading` shows spinner, `errorMessage != null` shows error + retry, else shows dashboard content. Dashboard content: `Column` with warning banner (conditional), `Row` (battery | speed | power), settings bar
- **`ui/screens/components/BatteryPanel.kt`**: SOC%, range, temp, charging state; colour changes by level
- **`ui/screens/components/SpeedDisplay.kt`**: speed in `displayLarge` white, km/h label, bike name
- **`ui/screens/components/PowerPanel.kt`**: HP, motor temp
- **`ui/screens/components/RideSettingsBar.kt`**: horizontal row -- power map, max power, regen, engine braking
- **`ui/screens/components/WarningBanner.kt`**: full-width, amber/red by severity
- Each composable gets `@Preview` functions. `DashboardScreen` gets previews for: content, loading, error, with-warning, low-battery -- ensuring every UiState field appears non-default in at least one preview

### Phase 7: Wiring

- **`MainActivity.kt`**: create repository, use case, factory, viewModel; collect state with `collectAsStateWithLifecycle`; dispatch `LoadDashboard` via `LaunchedEffect`
- **`strings.xml`**: all user-facing strings extracted, UK English, `global_` prefix for common ones
- Delete `ExampleUnitTest.kt` and `ExampleInstrumentedTest.kt`

## Acceptance Criteria (BDD)

### Scenario: ViewModel emits loading then content on LoadDashboard
```
GIVEN the use case will return a successful BikeInfo
WHEN the ViewModel receives LoadDashboard intent
THEN it first emits a state with isLoading = true
AND then emits a state with isLoading = false and all telemetry fields populated
```

### Scenario: ViewModel emits error state on failure
```
GIVEN the use case will return a failure Result
WHEN the ViewModel receives LoadDashboard intent
THEN it emits a state with errorMessage set to a user-friendly message
```

### Scenario: ViewModel reloads data on RetryLoad
```
GIVEN the ViewModel is in an error state
WHEN the ViewModel receives RetryLoad intent
THEN it clears the error and re-fetches the data
```

### Scenario: ViewModel populates warnings
```
GIVEN the use case returns a BikeInfo with active warnings
WHEN the ViewModel receives LoadDashboard intent
THEN the emitted state contains the warning list
```

### Scenario: Loading state shows spinner
```
GIVEN the DashboardScreen receives a UiState with isLoading = true
WHEN the screen renders
THEN a loading spinner is displayed
```

### Scenario: Error state shows message and retry button
```
GIVEN the DashboardScreen receives a UiState with an errorMessage
WHEN the screen renders
THEN the error message is displayed
AND a retry button is visible
```

### Scenario: Retry button emits RetryLoad intent
```
GIVEN the DashboardScreen is showing an error state
WHEN the user taps the retry button
THEN a RetryLoad intent is emitted
```

### Scenario: Content state shows speed, battery, and power
```
GIVEN the DashboardScreen receives a populated UiState
WHEN the screen renders
THEN the current speed is displayed prominently
AND the battery panel shows SOC%, range, and temperature
AND the power panel shows HP and motor temperature
```

### Scenario: Ride settings bar displays all settings
```
GIVEN the DashboardScreen receives a populated UiState
WHEN the screen renders
THEN the settings bar shows power map, max power, regen, and engine braking
```

### Scenario: Warning banner shows when warnings are present
```
GIVEN the DashboardScreen receives a UiState with warnings
WHEN the screen renders
THEN a warning banner is visible with the appropriate colour (amber or red)
```

### Scenario: Warning banner is hidden when no warnings
```
GIVEN the DashboardScreen receives a UiState with an empty warnings list
WHEN the screen renders
THEN no warning banner is displayed
```

### Scenario: Dashboard uses dark theme with Stark gold accent
```
GIVEN the app is launched
WHEN the dashboard renders
THEN the background is near-black (#0D0D0D)
AND accent elements use Stark gold (#C8A84E)
```

### Scenario: App launches and displays dashboard with telemetry
```
GIVEN the app is installed on a device
WHEN the app is launched
THEN the dashboard displays in landscape with speed, battery, power, settings, and bike name from the mock JSON
```
