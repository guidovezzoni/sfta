# 1.3 Dashboard UI

## User Story

**As a** rider glancing at the motorcycle dashboard,
**I want** a dark, high-contrast landscape display showing battery, power, ride settings, session stats, and warnings,
**So that** I can read my bike's telemetry at a glance while riding.

## Prerequisites

- **Story 1.2** (Data Model Null Safety) must be completed first — this story assumes all domain model fields are nullable.

## Scope

### Phase 1: Theme

- **`ui/theme/Color.kt`**: replace with dashboard palette — `DashboardBackground` (#0D0D0D), `DashboardSurface` (#1A1A1A), `StarkGold` (#C8A84E), semantic colours: `BatteryGreen`/`Amber`/`Red`, `WarningAmber`/`Red`, `PowerBlue`, `TextPrimary` (white), `TextSecondary` (light grey)
- **`ui/theme/Theme.kt`**: dark-only `darkColorScheme`, no dynamic colours, remove `darkTheme`/`dynamicColor` params
- **`ui/theme/Type.kt`**: add `displayLarge` (bold, for primary metrics like SOC% and HP), `titleLarge` (20sp for panel headers), `labelMedium` (12sp for labels)

### Phase 2: MVI Contract + ViewModel

- **`ui/state/DashboardUiState.kt`**: data class with all fields nullable (matching domain models) + UI-specific fields:
  - `isLoading: Boolean = false`
  - `errorMessage: String? = null`
  - `bikeName: String? = null`
  - Battery: `batteryChargePercent: Int?`, `batteryRangeKm: Int?`, `batteryTemperatureCelsius: Double?`, `chargingState: ChargingState?`
  - Power: `motorPowerHp: Double?`, `motorTemperatureCelsius: Double?`
  - Ride settings: `powerMap: PowerMap?`, `maxPowerHp: Int?`, `engineBrakingPercent: Int?`, `regenPercent: Int?`
  - Session: `sessionDurationSeconds: Long?`, `sessionDistanceKm: Double?`, `sessionMaxSpeedKmh: Double?`
  - Warnings: `warnings: List<WarningInfo>? = null`
- **`ui/intent/DashboardUiIntent.kt`**: `LoadDashboard`, `RetryLoad`
- **`ui/effect/DashboardUiEffect.kt`**: empty sealed class (structural placeholder)
- **`ui/viewmodel/DashboardViewModel.kt`**: `@HiltViewModel`, processes intents, calls use case, flattens domain model to UiState. Injected via `hiltViewModel()` — no manual factory needed.

### Phase 3: Composables

Layout (landscape, optimised for at-a-glance readability while riding):

```
┌─────────────────────────────────────────────────┐
│  ⚠ Warning Banner (conditional, full-width)     │
├──────────────────────┬──────────────────────────┤
│                      │                          │
│   BATTERY PANEL      │     POWER PANEL          │
│   73%  SOC           │     52.4 HP              │
│   38 km range        │     61.2°C motor         │
│   34.7°C             │                          │
│                      │                          │
├──────────────────────┴──────────────────────────┤
│  SESSION: 1h02m | 24.7 km | Max 94.1 km/h      │
├─────────────────────────────────────────────────┤
│  ENDURO | 80 HP max | 45% braking | 60% regen  │
└─────────────────────────────────────────────────┘
```

- **`ui/screens/DashboardScreen.kt`**: main screen composable — `when` block: `isLoading` shows spinner, `errorMessage != null` shows error + retry, else shows dashboard content. Dashboard content: `Column` with warning banner (conditional), primary `Row` (battery | power), session info row, settings bar
- **`ui/screens/components/BatteryPanel.kt`**: large SOC% as focal metric, range, temperature, charging state; colour changes by level (green ≥ 50%, amber 20–49%, red < 20%)
- **`ui/screens/components/PowerPanel.kt`**: large HP as focal metric, motor temperature
- **`ui/screens/components/SessionPanel.kt`**: horizontal row showing duration (formatted as Xh Xm), distance (km), max speed (km/h)
- **`ui/screens/components/RideSettingsBar.kt`**: horizontal row — power map, max power, regen, engine braking
- **`ui/screens/components/WarningBanner.kt`**: full-width, amber/red by severity
- **Null handling in all composables**: display "--" or equivalent placeholder when a value is null
- Each composable gets `@Preview` functions. `DashboardScreen` gets previews for: content, loading, error, with-warning, low-battery, null-fields — ensuring every UiState field appears non-default in at least one preview

### Phase 4: Wiring

- **`MainActivity.kt`**: use `hiltViewModel()` to obtain `DashboardViewModel`; collect state with `collectAsStateWithLifecycle`; dispatch `LoadDashboard` via `LaunchedEffect`
- **`strings.xml`**: all user-facing strings extracted, UK English, `global_` prefix for common ones
- Delete `ExampleUnitTest.kt` and `ExampleInstrumentedTest.kt`

## Acceptance Criteria (BDD)

### Scenario: ViewModel emits loading then content on LoadDashboard
```
GIVEN the use case will return a successful BikeInfo
WHEN the ViewModel receives LoadDashboard intent
THEN it first emits a state with isLoading = true
AND then emits a state with isLoading = false and all telemetry fields populated
```

### Scenario: ViewModel emits error state on failure
```
GIVEN the use case will return a failure Result
WHEN the ViewModel receives LoadDashboard intent
THEN it emits a state with errorMessage set to a user-friendly message
```

### Scenario: ViewModel reloads data on RetryLoad
```
GIVEN the ViewModel is in an error state
WHEN the ViewModel receives RetryLoad intent
THEN it clears the error and re-fetches the data
```

### Scenario: ViewModel populates warnings
```
GIVEN the use case returns a BikeInfo with active warnings
WHEN the ViewModel receives LoadDashboard intent
THEN the emitted state contains the warning list
```

### Scenario: ViewModel handles null domain fields
```
GIVEN the use case returns a BikeInfo with some null fields (e.g. battery = null)
WHEN the ViewModel receives LoadDashboard intent
THEN the emitted UiState has the corresponding fields as null
AND isLoading is false and errorMessage is null
```

### Scenario: Loading state shows spinner
```
GIVEN the DashboardScreen receives a UiState with isLoading = true
WHEN the screen renders
THEN a loading spinner is displayed
```

### Scenario: Error state shows message and retry button
```
GIVEN the DashboardScreen receives a UiState with an errorMessage
WHEN the screen renders
THEN the error message is displayed
AND a retry button is visible
```

### Scenario: Retry button emits RetryLoad intent
```
GIVEN the DashboardScreen is showing an error state
WHEN the user taps the retry button
THEN a RetryLoad intent is emitted
```

### Scenario: Content state shows battery and power panels
```
GIVEN the DashboardScreen receives a populated UiState
WHEN the screen renders
THEN the battery panel shows SOC%, range, and temperature
AND the power panel shows HP and motor temperature
```

### Scenario: Session info displays duration, distance, and max speed
```
GIVEN the DashboardScreen receives a populated UiState with session data
WHEN the screen renders
THEN the session panel shows formatted duration, distance in km, and max speed in km/h
```

### Scenario: Ride settings bar displays all settings
```
GIVEN the DashboardScreen receives a populated UiState
WHEN the screen renders
THEN the settings bar shows power map, max power, regen, and engine braking
```

### Scenario: Null fields display placeholder
```
GIVEN the DashboardScreen receives a UiState with some null fields
WHEN the screen renders
THEN null values are shown as "--" or an equivalent placeholder
AND the panel layout remains stable (no collapsed sections)
```

### Scenario: Warning banner shows when warnings are present
```
GIVEN the DashboardScreen receives a UiState with warnings
WHEN the screen renders
THEN a warning banner is visible with the appropriate colour (amber or red)
```

### Scenario: Warning banner is hidden when no warnings
```
GIVEN the DashboardScreen receives a UiState with an empty or null warnings list
WHEN the screen renders
THEN no warning banner is displayed
```

### Scenario: Dashboard uses dark theme with Stark gold accent
```
GIVEN the app is launched
WHEN the dashboard renders
THEN the background is near-black (#0D0D0D)
AND accent elements use Stark gold (#C8A84E)
```

### Scenario: App launches and displays dashboard with telemetry
```
GIVEN the app is installed on a device
WHEN the app is launched
THEN the dashboard displays in landscape with battery, power, session info, ride settings, and warnings from the mock JSON
```
