# 1.3 Dashboard UI

## User Story

**As a** rider glancing at the motorcycle dashboard,
**I want** a dark, high-contrast landscape display showing battery, power, ride settings, session stats, and warnings,
**So that** I can read my bike's telemetry at a glance while riding.

## Prerequisites

- **Story 1.2** (Data Model Null Safety) — completed. All domain model fields are nullable.
- **Story 2.3** (Introduce Hilt DI) — completed. `@HiltAndroidApp`, `@AndroidEntryPoint`, and `AppModule` are in place.

### Phase 2: MVI Contract + ViewModel

All new files under `app/src/main/java/com/guidovezzoni/sfta/ui/`.

#### `state/DashboardUiState.kt`

Flat data class that the ViewModel maps from the nested domain model (`BikeInfo`). All telemetry fields nullable to mirror the domain layer's null-safety model.

```kotlin
data class DashboardUiState(
    val isLoading: Boolean = false,
    val errorMessage: String? = null,
    val bikeName: String? = null,
    // Battery
    val batteryChargePercent: Int? = null,
    val batteryRangeKm: Int? = null,
    val batteryTemperatureCelsius: Double? = null,
    val chargingState: ChargingState? = null,
    // Power
    val motorPowerHp: Double? = null,
    val motorTemperatureCelsius: Double? = null,
    // Ride settings
    val powerMap: PowerMap? = null,
    val maxPowerHp: Int? = null,
    val engineBrakingPercent: Int? = null,
    val regenPercent: Int? = null,
    // Session
    val sessionDurationSeconds: Long? = null,
    val sessionDistanceKm: Double? = null,
    val sessionMaxSpeedKmh: Double? = null,
    // Warnings
    val warnings: List<WarningInfo>? = null,
)
```

Domain types referenced: `ChargingState`, `PowerMap`, `WarningInfo` (from `com.guidovezzoni.sfta.domain.model`).

#### `intent/DashboardUiIntent.kt`

```kotlin
sealed class DashboardUiIntent {
    data object LoadDashboard : DashboardUiIntent()
    data object RetryLoad : DashboardUiIntent()
}
```

#### `effect/DashboardUiEffect.kt`

```kotlin
sealed class DashboardUiEffect
```

Empty structural placeholder — no one-shot effects needed at this stage.

#### `viewmodel/DashboardViewModel.kt`

- Annotated with `@HiltViewModel`; injects `GetBikeInfoUseCase`.
- Exposes `uiState: StateFlow<DashboardUiState>` and `uiEffect: SharedFlow<DashboardUiEffect>`.
- `fun onIntent(intent: DashboardUiIntent)` dispatches to private handlers.
- `LoadDashboard` / `RetryLoad`:
  1. Emit `isLoading = true`.
  2. Call `getBikeInfoUseCase()`.
  3. On success: flatten `BikeInfo` fields into `DashboardUiState`.
  4. On failure: emit `errorMessage` with a user-friendly string (from `strings.xml`).
- Mapping logic (domain → UiState):
  - `bikeName` ← `bikeInfo.bike?.model` (optionally appending `variant`)
  - Battery fields ← `bikeInfo.battery?.stateOfChargePercent`, etc.
  - Motor fields ← `bikeInfo.motor?.powerHp`, etc.
  - Session fields ← `bikeInfo.session?.durationSeconds`, etc.
  - Ride settings ← `bikeInfo.rideSettings?.powerMap`, etc.
  - Warnings ← `bikeInfo.diagnostics?.warnings`

#### `DashboardScreen.kt`

- Receives `uiState: DashboardUiState` and `onIntent: (DashboardUiIntent) -> Unit = {}`.
- Top-level `when` block:
  - `isLoading` → `CircularProgressIndicator` centred on screen
  - `errorMessage != null` → error message text + "Retry" button (emits `RetryLoad`)
  - else → dashboard content layout
- Dashboard content: `Column` → optional `WarningBanner` → `Row(weight 1f, 1f)` for `BatteryPanel` | `PowerPanel` → `SessionPanel` → `RideSettingsBar`.
- `@Preview` coverage: content, loading, error, with-warnings, low-battery, null-fields — every `UiState` field appears non-default in at least one preview.

#### `components/BatteryPanel.kt`

- Parameters: `batteryChargePercent: Int?`, `batteryRangeKm: Int?`, `batteryTemperatureCelsius: Double?`, `chargingState: ChargingState?`, `modifier: Modifier = Modifier`.
- Large SOC% as focal metric using `displayLarge` typography.
- SOC colour logic: `BatteryGreen` ≥ 50%, `BatteryAmber` 20–49%, `BatteryRed` < 20%, `TextSecondary` if null.
- Range, temperature, and charging state as secondary metrics.
- Null values display "--".

#### `components/PowerPanel.kt`

- Parameters: `motorPowerHp: Double?`, `motorTemperatureCelsius: Double?`, `modifier: Modifier = Modifier`.
- Large HP as focal metric using `displayLarge` with `PowerBlue` accent.
- Motor temperature as secondary metric.

#### `components/SessionPanel.kt`

- Parameters: `sessionDurationSeconds: Long?`, `sessionDistanceKm: Double?`, `sessionMaxSpeedKmh: Double?`, `modifier: Modifier = Modifier`.
- Horizontal row: duration (formatted as `Xh Xm`), distance (`X.X km`), max speed (`X.X km/h`).

#### `components/RideSettingsBar.kt`

- Parameters: `powerMap: PowerMap?`, `maxPowerHp: Int?`, `engineBrakingPercent: Int?`, `regenPercent: Int?`, `modifier: Modifier = Modifier`.
- Horizontal row of settings labels.

#### `components/WarningBanner.kt`

- Parameters: `warnings: List<WarningInfo>`, `modifier: Modifier = Modifier`.
- Full-width banner; background colour based on highest severity (`WarningRed` for critical, `WarningAmber` for warning).

## Security Considerations

- **No sensitive data exposure:** All data is read from a local bundled JSON asset. No network calls, no user credentials, no PII.
- **No WebView or dynamic content loading:** Pure Compose UI, no injection vectors.

## Performance Considerations

- **Single data load:** The dashboard loads data once from a local file on `LoadDashboard`. No polling, no continuous updates.
- **Flat UiState:** The flattened state model avoids unnecessary recompositions from nested object changes.
- **Coroutine dispatching:** The repository already uses `Dispatchers.IO` for file reading. The ViewModel should launch in `viewModelScope` (defaults to `Dispatchers.Main`).

## GDPR & Sensitive Information

- No personal data is collected, stored, or transmitted.
- The JSON snapshot contains mock telemetry data only (battery, motor, session metrics).
- No analytics, tracking, or third-party SDKs introduced in this story.

## Non-Functional Requirements

- **Accessibility:** Use `contentDescription` on key metrics for screen reader support. Ensure sufficient colour contrast (dark background + white/gold text exceeds WCAG AA 4.5:1).
- **Orientation:** Dashboard is optimised for landscape viewing on a motorcycle mount. The layout should adapt gracefully if viewed in portrait.
- **Localisation:** All strings extracted to `strings.xml` in UK English, ready for future translation.
