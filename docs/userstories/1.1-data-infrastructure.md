# 1.1 Data Infrastructure

## User Story

**As a** developer setting up the rider dashboard,
**I want** the project infrastructure, data layer, and domain layer in place,
**So that** the app can load, parse, and map the bike telemetry snapshot ready for the UI to consume.

## Scope

### Phase 1: Project Infrastructure

- **Gradle (`gradle/libs.versions.toml`)**: add `kotlinx-serialization-json` (1.8.1), `androidx-lifecycle-viewmodel-compose` (2.10.0), `mockk` (1.14.4) testImplementation, `kotlinx-coroutines-test` (1.10.2) testImplementation, `kotlin-serialization` plugin using existing `kotlin` version ref
- **`build.gradle.kts` (project)**: add `kotlin-serialization` plugin `apply false`
- **`app/build.gradle.kts`**: apply serialization plugin + add 4 dependencies
- **Asset**: create `app/src/main/assets/bike_info_snapshot.json` -- copy from `docs/mocks/` with `current_speed_kmh: 47.3` added to `motor` object
- **Manifest**: add `android:screenOrientation="sensorLandscape"` to the activity

### Phase 2: Data Layer

- **`data/model/BikeInfoSnapshotDto.kt`**: all DTOs in one file, `@Serializable`, `@SerialName` for snake_case fields -- `BikeInfoSnapshotDto` (root), `BikeDto`, `BatteryDto`, `MotorDto` (with `currentSpeedKmh`), `RideSettingsDto`, `SessionDto`, `DiagnosticsDto`, `WarningDto`
- **`domain/repository/BikeInfoRepository.kt`**: interface with `suspend fun getBikeInfoSnapshot(): Result<BikeInfoSnapshotDto>`
- **`data/repository/LocalBikeInfoRepository.kt`**: reads from `AssetManager`, parses with `Json { ignoreUnknownKeys = true }`, wraps in `runCatching`, uses `Dispatchers.IO`

### Phase 3: Domain Layer

- **`domain/model/BikeInfo.kt`**: clean data classes (`BikeInfo`, `BatteryInfo`, `MotorInfo`, `RideSettingsInfo`, `WarningInfo`) + enums (`ChargingState`, `PowerMap`, `WarningSeverity`) each with `UNKNOWN` fallback
- **`domain/mapper/BikeInfoMapper.kt`**: `BikeInfoSnapshotDto.toDomain()` extension
- **`domain/usecase/GetBikeInfoUseCase.kt`**: calls repository, maps result

## Acceptance Criteria (BDD)

### Scenario: Project compiles with new dependencies
```
GIVEN the serialization and lifecycle dependencies have been added
WHEN I run ./gradlew assembleDebug
THEN the project compiles without errors
```

### Scenario: JSON asset is present and valid
```
GIVEN the bike_info_snapshot.json file is in app/src/main/assets/
WHEN the file is parsed with Kotlinx Serialization
THEN it deserialises into BikeInfoSnapshotDto without errors
AND the motor object contains current_speed_kmh = 47.3
```

### Scenario: Repository reads valid JSON successfully
```
GIVEN a valid bike_info_snapshot.json is available via AssetManager
WHEN getBikeInfoSnapshot() is called
THEN it returns Result.success containing the parsed BikeInfoSnapshotDto
```

### Scenario: Repository handles missing file gracefully
```
GIVEN the asset file does not exist
WHEN getBikeInfoSnapshot() is called
THEN it returns Result.failure with an appropriate exception
```

### Scenario: Repository handles malformed JSON gracefully
```
GIVEN the asset file contains invalid JSON
WHEN getBikeInfoSnapshot() is called
THEN it returns Result.failure with a serialisation exception
```

### Scenario: Mapper converts DTO to domain model completely
```
GIVEN a fully populated BikeInfoSnapshotDto
WHEN toDomain() is called
THEN a BikeInfo domain model is returned with all fields correctly mapped
```

### Scenario: Mapper handles unknown enum values with fallback
```
GIVEN a BikeInfoSnapshotDto with unrecognised enum string values
WHEN toDomain() is called
THEN the corresponding enum fields fall back to UNKNOWN
```

### Scenario: Mapper handles empty warnings list
```
GIVEN a BikeInfoSnapshotDto with an empty warnings list
WHEN toDomain() is called
THEN the BikeInfo.warnings list is empty
```

### Scenario: Use case returns mapped domain model on success
```
GIVEN the repository returns a successful Result
WHEN GetBikeInfoUseCase is invoked
THEN it returns Result.success containing the mapped BikeInfo domain model
```

### Scenario: Use case propagates failure from repository
```
GIVEN the repository returns a failure Result
WHEN GetBikeInfoUseCase is invoked
THEN it returns the same Result.failure
```

### Scenario: App is locked to landscape orientation
```
GIVEN the AndroidManifest has screenOrientation="sensorLandscape" on the activity
WHEN the app is launched on a device
THEN it displays only in landscape orientation
```
