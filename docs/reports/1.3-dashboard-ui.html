User Story

1.3 Dashboard UI — As a rider glancing at the motorcycle dashboard, I want a dark, high-contrast landscape display showing battery, power, ride settings, session stats, and warnings, so that I can read my bike's telemetry at a glance while riding.

Branch

Created branch feature/1.3-dashboard-ui from main.

Refinement Summary

The original user story was already well-structured with clear phases and BDD scenarios. The refinement added:

  • Fully qualified file paths using the actual package com.guidovezzoni.sfta
  • Exact colour hex values and typography specs in tabular format for the theme
  • Complete DashboardUiState data class definition with domain type references (ChargingState, PowerMap, WarningInfo)
  • Detailed domain-to-UiState mapping logic for the ViewModel
  • Composable parameter signatures for all components
  • Complete strings.xml key-value table
  • Expanded unit test and Compose UI test tables with specific test case names following GIVEN/WHEN/THEN pattern
  • Security, performance, GDPR, and non-functional requirements analysis (all low risk)
  • Definition of Done checklist (11 items)

Change Proposed

3 June 2026

Source: opsx:propose

Exploration Findings

  • All prerequisites confirmed complete: Clean Architecture layers, Hilt DI, domain models with nullable fields, GetBikeInfoUseCase
  • Identified two missing dependencies: hilt-navigation-compose (for hiltViewModel()) and lifecycle-runtime-compose (for collectAsStateWithLifecycle())
  • App already locked to sensorLandscape in manifest
  • Current theme uses default Material purple/pink palette with dynamic colours and light/dark toggle — full replacement required
  • User story is highly detailed with exact field definitions, colour hex values, typography specs, composable signatures, strings.xml entries, and 15 BDD scenarios

Questions and Clarifications

No ambiguities found — the user story serves as a complete technical specification. All design decisions were made during exploration:

  • Theme function name StarkFutureTechnicalAssessmentTheme retained (internals changed to dark-only)
  • Flat DashboardUiState chosen over nested sub-states
  • Domain enum types (ChargingState, PowerMap) referenced directly in UiState
  • Battery colour thresholds as inline composable logic, not ViewModel mapping
