User Story

1.2 Data Model Null Safety — Make all DTO and domain model fields nullable to handle incomplete telemetry gracefully, and remove the obsolete currentSpeedKmh field.

Branch

Story Refinement Summary

  • Added explicit enum nullability semantics: null = no data from sensor; UNKNOWN = unrecognised value received.
  • Added detailed mapper rules: null sub-objects, null enums, null timestamps, and null lists each have specific mapping behaviour.
  • Added non-functional analysis: no security, performance, GDPR, or forward-compatibility concerns.
  • Added a Definition of Done checklist with 7 concrete items.
  • Expanded test plan with 5 new test cases for null-handling scenarios and explicit fixture update instructions for 3 existing test files.
  • Scope covers 20 files: 17 production source files and 3 test files.
