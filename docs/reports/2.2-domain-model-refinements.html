Change Proposed

3 Jun 2026

Source: opsx:propose

Exploration Findings

  • BikeInfo.timestamp is currently a raw String — needs to become kotlinx.datetime.Instant
  • DiagnosticsDto.faultCodes is already parsed from JSON but silently dropped in the mapper — only warnings is mapped to DiagnosticsInfo
  • The JSON asset has fault_codes: [] (empty), while the test fixture uses listOf("F001") — both cases are covered
  • The spec (bike-telemetry-data/spec.md) explicitly states to exclude faultCodes — this needs updating
  • Repository test doesn't currently assert timestamp value — needs a new assertion
  • No ambiguities or blockers identified — the user story is fully specified
